    How to prove 2 < 3 in Agda

    I'm currently playing Euclidean geometry with Agda, and sometimes, I need to give an evidence for relations like:

    $$ 1 + 1 > 1 $$

    (This one is used in proving there have to be two different intersection points for two 1 radius circles when proving Proposition 1 in Στοιχεῖα)

    In fact, when met up with this problem first time, I felt hard to prove this "obvious" proposition1, especially when these 1s are actually 1ℚ. I tried hard to find out a tool to do this in the stdlib and finally I get to those Decidable related things.

    How to prove 2 < 3

    Let's start small, how can we give an evidence for 2 ≤ 3, when 2 and 3 are ℕs in Agda?

    View the definition of ≤ in Data.Nat.Base:

    data _≤_ : Rel ℕ 0ℓ where
  z≤n : ∀ {n}                 → zero  ≤ n
  s≤s : ∀ {m n} (m≤n : m ≤ n) → suc m ≤ suc n

    So z≤n is the evidence for 0 ≤ n, and s≤s is the evidence for m ≤ n when m ≤ n is given. So we can prove 2 ≤ 3 by using s≤s twice on z≤n:

    2≤3 : 2 ≤ 3
2≤3 = s≤s (s≤s z≤n)

    Push it further

    This seems easy, right? But what if we want to prove, say 99 ≤ 100? It's insane to use s≤s 99 times, right?

    We can also find another version of ≤, which returns Bools in Data.Nat:

    _≤ᵇ_ : (m n : ℕ) → Bool
zero  ≤ᵇ n = true
suc m ≤ᵇ n = m <ᵇ n

    where

    _<ᵇ_ : Nat → Nat → Bool
_     <ᵇ zero  = false
zero  <ᵇ suc _ = true
suc n <ᵇ suc m = n < m

    So we know the value of term 99 ≤ᵇ 100 is (≡) true, the computation process of≤ᵇseems somehow like, or at least related to an evidence of99 \leq 100.

    And here's where Decidable things can help.

    The proof of 99 ≤ 100 can be written like:

    99≤100 : 99 ≤ 100
99≤100 = ≤ᵇ⇒≤ 99 100 tt

    So what is ≤ᵇ⇒≤? Why we need to pass a tt here? Let's read the definition of ≤ᵇ⇒≤.

    ≤ᵇ⇒≤ : ∀ m n → T (m ≤ᵇ n) → m ≤ n
≤ᵇ⇒≤ zero    n m≤n = z≤n
≤ᵇ⇒≤ (suc m) n m≤n = <ᵇ⇒< m n m≤n

    where

    <⇒<ᵇ : ∀ {m n} → m < n → T (m <ᵇ n)
<⇒<ᵇ (s≤s z≤n)       = tt
<⇒<ᵇ (s≤s (s≤s m<n)) = <⇒<ᵇ (s≤s m<n)

    and

    T : Bool → Set
T true  = ⊤
T false = ⊥

    So for ≤ᵇ⇒≤, we put the numbers we want to compare in, and then feed a tt to the type checker, which will check whether T (m ≤ᵇ n) is tt. If it is, then we get the evidence of m ≤ n, otherwise the type checker will complain.

    All other details in ≤ᵇ⇒≤ are trivial, just naive pattern matchings.

    Decidable

    You may see another kind of ≤ : ≤?, what's this for? Let's view its definition.

    _≤?_ : Decidable _≤_
m ≤? n = map′ (≤ᵇ⇒≤ m n) ≤⇒≤ᵇ (T? (m ≤ᵇ n))

    There're several things we are not familiar with in this definition, let's check them one by one.

    Decidable

    Decidable : REL A B ℓ → Set _
Decidable _∼_ = ∀ x y → Dec (x ∼ y)

    REL here is an abstract of binary relations between two types (in another word, Heterogeneous binary relations). So Decidable means for each x and y in A and B, we have Dec (x op y).

    And the real interesting part is Dec, which is a record type:

    record Dec {p} (P : Set p) : Set p where
  constructor _because_
  field
    does  : Bool
    proof : Reflects P does

    data Reflects {p} (P : Set p) : Bool → Set p where
  ofʸ : ( p :   P) → Reflects P true
  ofⁿ : (¬p : ¬ P) → Reflects P false
``

`Reflects` type means the prop `P`'s truth value is reflected (or, in another word, decided) by the `Bool` value, it has two constructors, `ofʸ` means `P` is true, and `ofⁿ` means `P` is false.

`Reflects P b` is equivalent to `if b then P else ¬ P`, and these two forms can be casted back and forth with these two functions;

```agda
of : ∀ {b} → if b then P else ¬ P → Reflects P b
of {b = false} ¬p = ofⁿ ¬p
of {b = true }  p = ofʸ p

invert : ∀ {b} → Reflects P b → if b then P else ¬ P
invert (ofʸ  p) = p
invert (ofⁿ ¬p) = ¬p

    And Dec is just a combination of a Bool and the proof that a prop P's truth value is reflected by the Bool.

    There also exists two patterns:

    pattern yes p =  true because ofʸ  p
pattern no ¬p = false because ofⁿ ¬p

    which can be understand from their literal meaning.

    Now we know Decidable means for each x and y in A and B, there is a Bool which reflects x op y.

    T?

    T? : Relation.Unary.Decidable T
does  (T? b) = b
proof (T? true ) = ofʸ _
proof (T? false) = ofⁿ λ()

    Relation.Unary.Decidable is the unary version of Decidable we mentioned above (Decidable P = ∀ x → Dec (P x)).

    So T? is a function which takes a Bool and returns an instance of Decidable, which is a term that shows some certain prop's truth value is reflected by that Bool

    ≤⇒≤ᵇ

    Just a reverse of ≤ᵇ⇒≤.

    ≤⇒≤ᵇ : ∀ {m n} → m ≤ n → T (m ≤ᵇ n)
≤⇒≤ᵇ z≤n         = tt
≤⇒≤ᵇ m≤n@(s≤s _) = <⇒<ᵇ m≤n

    map′

    map′ : (P → Q) → (Q → P) → Dec P → Dec Q
does  (map′ P→Q Q→P p?)                   = does p?
proof (map′ P→Q Q→P (true  because  [p])) = ofʸ (P→Q (invert [p]))
proof (map′ P→Q Q→P (false because [¬p])) = ofⁿ (invert [¬p] ∘ Q→P)

    So map′ means if two props are in "iff" relation, and one of them is Dec, we know the other is also Dec.

    Now we can come back to ≤? and understand what is:

    _≤?_ : Decidable _≤_
m ≤? n = map′ (≤ᵇ⇒≤ m n) ≤⇒≤ᵇ (T? (m ≤ᵇ n))

    Since ≤ᵇ⇒≤ and ≤⇒≤ᵇ are inverse function of each other, and T? (m ≤ᵇ n) is a Dec instance for m ≤ᵇ n, so we can get a Dec instance for m ≤ n.

    So what's the benefit of adopting ≤??

    Well, we can get both Bool value out of a Dec:

    Dec.does (2 ≤? 1)
-- or
isYes (2 ≤? 1)
-- or
⌊ 2 ≤? 1 ⌋

    The beneficial of isYes and the ⌊⌋ operator is they can keep the typing information, while Dec.does is a naive Bool.

    And we can also get the evidence out with a function named toWitness:

    99≤100' : 99 ≤ 100
99≤100' = toWitness {Q = 99 ≤? 100} tt

    where:

    toWitness : {Q : Dec P} → True Q → P
toWitness {Q = true  because [p]} _  = invert [p]
toWitness {Q = false because  _ } ()

    where

    True : Dec P → Set
True Q = T (isYes Q)

    We can see Decidable can be used in both places where Bool or an evidence is needed. So it is always preferred to use Decidable instead of Bool when we can.

    Usecases

    Take a constructor of ℚ for an example:

    _/_ : (n : ℤ) (d : ℕ) → {d≢0 : d ≢0} → ℚ
(+ n      / d) {d≢0} =   normalize n       d {d≢0}
(-[1+ n ] / d) {d≢0} = - normalize (suc n) d {d≢0}

    Here {d≢0 : d ≢0} requires denominator of a rational number should not be zero.

    ≢0 is defined as

    n ≢0 = False (n ℕ.≟ 0)

    And ℕ.≟ is very similar with ≤?:

    _≟_ : Decidable {A = ℕ} _≡_
m ≟ n = map′ (≡ᵇ⇒≡ m n) (≡⇒≡ᵇ m n) (T? (m ≡ᵇ n))

    By requiring an ≢0 instance (instead of requiring a "real" evidence of n ≢ 0), we can omit the proof when we are using number literals:

    half : ℚ
half = 1ℤ / 2
